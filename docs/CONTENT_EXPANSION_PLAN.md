## 更新规则

- 开始动笔前：把对应条目从 `TODO` 改成 `DOING`
- 内容写入文件后：改成 `DONE`
- 跑完检查并人工抽查通过后：改成 `CHECKED`
- 任一条目结束当轮后，必须同步更新本文件，不留悬空状态

```bash
node -e "const fs=require('fs'); ['data/details-ancient.js','data/details-classical.js','data/details-islamic.js','data/details-medieval.js','data/details-ottoman.js','data/details-modern.js','data/details-contemporary.js','data/conflicts.js','js/app.js'].forEach(f=>new Function(fs.readFileSync(f,'utf8'))); console.log('syntax ok');"
```

每轮至少做三项检查：

- 条目 key 仍能在对应文件中准确搜到
- 详情卡不再是简略摘要，已经补进完整结构
- 本文档状态已同步更新

当代时间敏感条目额外要求：

- 标明复核日期
- 把已知事实与解释性判断尽量分开
